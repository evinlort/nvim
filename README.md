# Neovim config (Python-focused, Neovim 0.11+)

This repository contains a modular Neovim configuration optimized for Python work:
- LSP: basedpyright (strict) + ruff
- Formatting: Black via none-ls
- Linting: mypy via nvim-lint
- Navigation: Telescope + NvimTree + Aerial outline
- UI: Catppuccin + Lualine + Trouble + Which-key
- Python venv handling: venv-selector + fallback venv search
- Terminal REPL: IPython via toggleterm

> Note: `linux-cultist/venv-selector.nvim` relies on Neovim 0.11 LSP APIs (`vim.lsp.config`, `vim.lsp.enable`), so Neovim 0.11+ is required.  (See plugin notes.)

## Load Order

`init.lua` loads modules in this exact order:

1. `core.python_host` (sets `g:python3_host_prog` if possible)
2. Disables netrw (`vim.g.loaded_netrw`, `vim.g.loaded_netrwPlugin`) for NvimTree
3. `core.options`
4. `core.leader` (leader is `\`)
5. `core.autocmds`
6. `core.diagnostic_config` (global `vim.diagnostic.config`)
7. `core.lazy` (bootstraps lazy.nvim and loads `lua/plugins/init.lua`)

## Core Behavior

### Options (`lua/core/options.lua`)
- listchars enabled (tab/trailing/NBSP)
- line numbers enabled
- mouse enabled
- `switchbuf` uses tabs
- tags path includes local `./tags,tags;` and a cache tags directory

### Leader (`lua/core/leader.lua`)
- Leader key: `\`

### Autocommands (`lua/core/autocmds.lua`)
- clears search highlight ~2 seconds after leaving `/` or `?` search
- copies default yank register (`"`) into system clipboard register (`+`) after yank

### Diagnostics (`lua/core/diagnostic_config.lua`)
- disables inline virtual text diagnostics
- rounded floating diagnostics with:
  - `source = true`
  - header `"Troubles:"`
  - prefix `"● "`

### Python host (`lua/core/python_host.lua`)
Sets Neovim python host interpreter in this priority:
1. `$NVIM_PYTHON3_HOST_PROG` / `$PYTHON3_HOST_PROG` (if executable)
2. `python3.12`
3. `python3`

## LSP Details (Neovim 0.11 style)

LSP servers are configured in `lua/plugins/lsp_servers.lua` using `vim.lsp.config[...]` and enabled via `vim.lsp.enable(...)`.

### Servers
- `basedpyright`:
  - `typeCheckingMode = "strict"`
  - several `diagnosticSeverityOverrides`
  - `pythonPath` is resolved from `core.python_env`
  - `extraPaths` adds `<project_root>/src` if it exists
- `ruff`:
  - `init_options.settings.args = {}`

### Capabilities
- `cmp_nvim_lsp.default_capabilities()` is used and passed into basedpyright config.

### Python environment handling
Single source of truth: `lua/core/python_env.lua`.
Priority:
1. If `venv-selector.nvim` is active, it provides python/venv paths via `lua/core/venv_selector_adapter.lua`.
2. Otherwise, fallback upward search for `.venv`, `venv`, `.env` directories.

Used by:
- LSP `pythonPath`
- toggleterm IPython mappings

## Formatting (Black)

Configured in `lua/plugins/formatting.lua` via none-ls:
- Black with `--line-length=100`
- Keymap: `<leader>fo` (formats using client name `"null-ls"`)

## Linting (mypy)

Configured in `lua/plugins/lint.lua`:
- Only `mypy` for Python
- Auto-run on: `BufEnter`, `BufWritePost`
- Manual trigger: `<leader>l`

## Explorer (NvimTree)

Configured in `lua/plugins/explorer.lua`:
- netrw disabled in `init.lua`
- dotfiles shown
- git integration enabled
- does not quit on open
- buffer-local mappings in NvimTree:
  - `t` / `<CR>` open node in a new tab
  - `c` create file
  - `r` rename
  - `d` delete
  - `<2-LeftMouse>` open by double-click (tab)

## Telescope

Configured in `lua/plugins/search.lua`:
- `<C-t>` opens selected entry in a new tab (in Telescope prompt)

## Statusline (Lualine + Navic)

Configured in `lua/plugins/statusline.lua` + `lua/plugins/lsp-navic.lua`:
- filename with `path = 1`
- breadcrumbs from navic when available

## Comments + minimap (mini.nvim)

Configured in `lua/plugins/mini.lua`:
- commenting:
  - `gcc` comments the current line
  - `<leader>cc` comments the current line (normal) / selection (visual)
- minimap (`mini.map`):
  - `<leader>mo` toggle
  - `<leader>mm` open
  - `<leader>mc` close
  - `<leader>mf` focus/unfocus
